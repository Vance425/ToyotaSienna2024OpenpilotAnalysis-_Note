# `n1 / n3` Weak Patterns

## 目標

在兩段最乾淨的 steady 樣本中，檢查 `0x116 tail` 的：

- `n1`
- `n3`

是否存在弱規律。

樣本：

- `184921 / phase 27 00 / steady`
- `185451 / phase 09 0a / steady`

## 樣本 1: `184921 / 27 00 / steady`

tail:

- `2321fa94`
- `6f8809ce`
- `a63aa1db`
- `e86fcf4c`

### `n1`

- 值：`3 -> f -> 6 -> 8`
- delta：`+12 +7 +2`

### `n3`

- 值：`1 -> 8 -> a -> f`
- delta：`+7 +2 +5`

判讀：

- `n1` 沒有主骨架
- `n3` 也不像乾淨 counter
- 但 `n3` 的 `+2`、`+5` 這種局部步進值得留意

## 樣本 2: `185451 / 09 0a / steady`

tail:

- `a1669e30`
- `e27e3102`
- `2e097cf8`
- `6c3a68ef`
- `abdf5f55`

### `n1`

- 值：`1 -> 2 -> e -> c -> b`
- delta：`+1 +12 +14 +15`

### `n3`

- 值：`6 -> e -> 9 -> a -> f`
- delta：`+8 +11 +1 +5`

判讀：

- `n1` 仍然沒有穩定骨架
- `n3` 再次出現 `+1`、`+5` 這種局部片段

## 合併結論

### `n1`

目前比較像：

- 弱規律很少的過渡位

它會動，但還看不出固定轉移集合。

### `n3`

目前比較像：

- 帶少量局部規律的動態過渡位

它沒有 `n0` 那樣的主骨架，但在不同樣本裡都出現了一些相近的局部步進片段。

## 目前最實用的判讀

如果要排序 tail 的可建模性，現在大致是：

1. `n0`
2. `n3`
3. `n1`
4. `n4-n7` 最像 auth-heavy

## 對後續的意義

`n1 / n3` 目前不適合當主軸，但適合當輔助訊號：

- `n0` 給主要 rolling 骨架
- `n3` 提供局部過渡感
- `n1` 先保留，不過度解讀
