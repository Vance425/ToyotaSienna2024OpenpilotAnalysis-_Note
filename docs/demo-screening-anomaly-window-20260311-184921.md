## 第三輪示範結論

這個窗口最值得注意的，不是單一 ID 本身，而是：

- `0x610` 提供了事件中心點
- `0x131` 出現像 reset / restart 的 prefix 行為
- `0x116` 出現明確 phase 遞進
- `0x260` 出現整體模式切換
- `0x115` 同步跟著變
- `0xD5` 不變

這是一種很像「同一個狀態邊界，同時影響多條候選線」的窗口。

## 目前最有利的推進方式

之後如果要找更像 `comma 3X` 失敗的窗口，就要優先找這種特徵：

1. `0x610` 或其他參考線出現事件點
2. `0xD5` 沒有一起變
3. `0x131` 有 reset / restart 感
4. `0x116` 有 phase 跳變
5. `0x260` 有整體模式切換

## 這一輪後的優先觀察順序
