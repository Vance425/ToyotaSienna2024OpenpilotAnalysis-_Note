# Three-Axis Alignment

## 目的

把三條最重要的軸放在一起看：

- `0x116.b0-b1`: phase
- `0x131.b1 / b3`: boundary + state base
- `0x260.b2 / b4`: family sync

要回答的問題是：

`0x116` 的 phase change，是不是總發生在 `0x131 / 0x260` 的 state 邊界附近。

## 樣本 1: `184921`

檔案：

- [toyota_seg_IGN_ON_20260311_184921_000.ndjson](/D:/Codex/toyota-sienna-tsk-analysis/analysis-input/raw_can_logs/toyota_seg_IGN_ON_20260311_184921_000.ndjson)

### 三條軸摘要

- `0x131`: `37/3e -> 38/3e -> 3b/3d -> 3d/3d -> 3e/3d -> 00/3d -> 03/3d -> 04/3e -> 06/3e -> 07/3e -> 09/3f -> 0a/3f -> 0c/3f -> 0d/40 -> 0f/41`
- `0x260`: `42/3e -> 42/3d -> 44/3d -> 45/3d -> 47/3d -> 49/3d -> 4c/3d -> 4f/3e -> 53/3e -> 55/3e -> 57/3e -> 59/3f -> 5b/3f -> 5c/40 -> 5c/41 -> 5b/42`
- `0x116`: `24/00 -> 25/00 -> 26/00 -> 27/00 -> 28/00 -> 29/00`

### 時序觀察

先發生的是 `0x131` reset：

- `003e003d -> 0000003d`

接著 `0x116` phase 從：

- `25 00 -> 26 00 -> 27 00`

而 `0x260` 同時從：

- `47/3d -> 49/3d -> 4c/3d -> 4f/3e`

一路推進到新的 family 狀態。

### 這段的結論

這不是三條線各做各的，而比較像：

1. `0x131` 宣告跨邊界
2. `0x260` 進入新 family state
3. `0x116` 在邊界附近切新 phase

## 樣本 2: `185451`

檔案：

- [toyota_seg_IGN_ON_20260311_185451_000.ndjson](/D:/Codex/toyota-sienna-tsk-analysis/analysis-input/raw_can_logs/toyota_seg_IGN_ON_20260311_185451_000.ndjson)

### 三條軸摘要

- `0x131`: `3c/d3 -> 3e/c7 -> 3f/bc -> 01/a8 -> 04/9a -> 05/95 -> 07/91 -> 08/90 -> 0a/8d -> 0b/8d -> 0d/8d -> 0e/8d -> 10/8b -> 11/8b -> 13/8a`
- `0x260`: `f2/c7 -> fb/bc -> fc/b2 -> fd/9a -> fe/95 -> 01/91 -> ef/90 -> 04/8d -> f6/8d -> e6/8b`
- `0x116`: `09/09 -> 09/0a -> 0a/0c -> 0d/0e`

### 時序觀察

一開始 `0x131` 先從：

- `3c/d3 -> 3e/c7 -> 3f/bc`

翻到：

- `01/a8 -> 04/9a -> 05/95`

同時 `0x260` 的 state base 也跟著降到：

- `c7 -> bc -> b2 -> 9a -> 95 -> 91 -> 90 -> 8d`

而 `0x116` 則在這段切換帶中：

- 長時間停在 `09/0a`
- 經過 anchor 後切到 `0a/0c`
- 再切到 `0d/0e`

### 這段的結論

這段比 `184921` 更像「多階段狀態切換」，不是單次跳點：

1. `0x131` 先跨邊界
2. `0x260` 把新的 family base 展開
3. `0x116` 在這個新 base 上切 phase

## 合併結論

目前最合理的事件模型是：

1. `0x131` 提供邊界訊號
2. `0x260` 提供 family state 同步
3. `0x116` 在邊界附近切 protected phase

也就是說，`0x116` 的 phase 並不是孤立事件，而是在 `0x131 + 0x260` 定義出的 family 邊界上切換。

## 目前最可用的工作判準

如果之後看新 log，要判定某個窗口是不是高價值樣本，可以用這三軸規則：

1. `0x131` 前段出現 reset-like 或 boundary-like 改變
2. `0x260` 的 family state 軸同步改變
3. `0x116` 在同一帶內切 phase
4. `0xD5` 保持不動

滿足這四件事的窗口，優先度最高。
