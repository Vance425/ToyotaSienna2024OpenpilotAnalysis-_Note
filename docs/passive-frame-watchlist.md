# 被動觀察名單

## Tier 1

### bus 0 / 0x116

- 目前最強候選
- 優先看 bytes `4,5,6,7`
- 觀察重點：tail 是否在狀態切換或失敗時序中出現 reset / 停頓 / 模式改變

### bus 0 / 0x115

- 根據 `v8.1` 升級為 Tier 1
- 優先看 bytes `4,5,6,7`
- 觀察重點：是否與 `0x116 / 0x131` 呈現同步或互補變化

### bus 0 / 0x131

- 根據 `v8.1` 升級為 Tier 1
- 優先看 bytes `4,5,6,7`
- 觀察重點：是否在狀態切換與失敗窗口中與 `0x115 / 0x116` 成群出現

## Tier 2

### bus 0 / 0x260

- 高熵 / 切換敏感候選
- 優先看 bytes `6,7`
- 觀察重點：payload 大幅變化時，tail 是否仍保持類似 counter/auth 行為

### bus 0 / 0x177

- 保留觀察
- 之前 `secoc_cracker_v5` 分數很高
- 但在 `v8.1` 這批 focus 裡沒有像 `0x115 / 0x116 / 0x131` 那麼突出

## Tier 3

### bus 0 / 0xD8

- 主要當速度 / 動態參考訊號
- 不當成 `TSK` 主候選
- 用來對照車輛是否真的在移動或切換狀態

### bus 1

- 尚未定出明確單一 ID
- 先從狀態切換與 `comma 3X` 失敗窗口中找出異常突出的 frame

## 每次觀察都要記錄

- 場景
- bus
- ID
- 變化時間點
- 變化前後的 payload
- 是否與 `comma 3X` 失敗時間重疊
