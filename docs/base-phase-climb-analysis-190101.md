### 判讀

這不是隨機跳動，而是很完整的 phase ramp。

它看起來像：

- 離開基底態
- 進入某個有階段的運作態
- 再往更高 family/phase 區間推進

## 之後的基底離開樣式

後面還有多個 `00 00` 基底段，離開方式包括：

- `00 00 -> 00 03`
- `00 00 -> 00 02`
- `00 00 -> 00 01`

代表：

- 離開基底態不是單一固定起點
- 但幾乎都先從低位 phase 開始往上爬

## 重要觀察

### 1. `00 00` 不是終點，而是基底態

前面 failure proxy 已經看到 session 尾端會收斂到 `00 00`。

這裡又看到：

- 新 session 或新事件帶，也會從 `00 00` 往上爬

所以 `00 00` 比較像：

- protected family 的 base phase

### 2. 爬升帶有長有短

有些基底離開只到：

- `00 01`
- `00 02`
- `00 03`

但第一個完整樣本一路爬到：

- `1e 1e`

這說明：

- 不同事件帶的深度不同
- 並不是每次離開基底都會爬到很高 phase

### 3. 這是目前最值得追的入口

如果之後要找「安全門打開」的候選模式，最值得比對的不是單點 frame，而是：

- `00 00` 離開後的 phase ramp 深度和形狀

## 目前最有價值的工作假設

### 假設 A

`00 00` 是 base phase。

### 假設 B

離開 `00 00` 的爬升帶深度，可能反映不同等級的 session/state 開啟程度。

### 假設 C

若未來能抓到真正 `comma 3X` 失敗或成功附近的 log，最值得比的是：

- phase ramp 是否比一般 driving 更深
- 或是否在某個特定 phase 深度停住
