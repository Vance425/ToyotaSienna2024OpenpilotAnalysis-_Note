## 1. 进入条件

### family pair 先稳定到 `fff4|fff4`

在 phase 还停留 `00 00` 时，family pair 就已经先稳定：

- `f131 = fff4`
- `f260 = fff4`

对应状态前缀：

- `0x131`: `0038fff4 -> 003bfff4 -> 003dfff4`
- `0x260`: `000008fff4 -> 000002fff4 -> 00ffebfff4 -> 00ffe6fff4`

### `0x116` 仍在 base phase，但 `n0` 已经继续走

在 `00 00` 期间：

- `n0`: `f -> 3 -> 4 -> 8 -> c`

这里已经能看到：

- `+4` backbone
- 外加一次 `+1` 轻微修正

### 进入触发点

随后 phase 开始离开 `00 00`：

- `00 00 -> 04 0b -> 10 1e -> 20 31 -> 2A 3A`

其中关键起跳是：

- `00 00 -> 04 0b` 对应 `n0: c -> 4` (`+8`)

这符合当前对 boundary re-seed 的定义。

## 2. 平台条件

### 平台上的固定条件

平台期间共同满足：

- `f131 = fff4`
- `f260 = fff4`

`n0` 行为：

- `2A 3A -> 43 48`: `0 -> c` (`+12`)

### 判读

- 平台建立后，`+4` 很快重新出现
- `fff4|fff4` 是最稳定的 family pair

## 3. 退出条件

### family pair 开始松动

平台退出时，最先变化的是：

- `f260: fff4 -> fff0`

此时 `f131` 仍是：

- `fff4`

对应 phase：

- `3E 3C`

### 之后继续下退

退出后 family pair 沿走廊继续下降：

- `fff4|fff0`
- `fff0|ffee`
- `fff0|ffeb`
- `ffeb|ffea`

phase 也同步下降：

- `3E 3C`
- `3C 3A`
- `3B 3B`

### `n0` 的退出信号

退出瞬间：

- `42 40 -> 3E 3C`: `4 -> 5` (`+1`)

这和当前模型一致：

- `+1` 更像轻微修正 / 平台松动信号

## 当前最实用的三段式模板

### Entry

1. `0x131 / 0x260` 先稳定到 `fff4|fff4`
2. `0x116` 仍在 `00 00`
3. `n0` 在 base phase 内继续沿 backbone 前进
4. 用一次 `+8` re-seed 跳离 `00 00`

### Plateau

1. `0x116` 推到 `43+ / 44+`
2. `0x131 / 0x260` 保持 `fff4|fff4`
3. `n0` 重新回到 `+4` 主骨架

### Exit

1. `0x260` 先从 `fff4` 松到 `fff0`
2. `0x116` 从 `42 40` 降到 `3E 3C`
3. `n0` 出现 `+1` 轻微修正

## 这对后续的意义

之后看新 log，不需要只盯“有没有 `43 48`”。

更好的做法是看完整生命周期：

- 先有没有 `fff4|fff4 + 00 00`
- 再有没有 `+8` 离开 base phase
- 然后有没有高位平台
- 最后是否按 `fff4|fff0` 方式退出
