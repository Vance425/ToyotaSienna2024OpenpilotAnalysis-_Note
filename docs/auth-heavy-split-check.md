# Auth-Heavy Split Check

## 目標

檢查 `0x116 tail` 的 `n4-n7` 是否完全高熵，還是還能再拆出相對穩一些的子區。

樣本：

- `184921 / 27 00 / steady`
- `185451 / 09 0a / steady`

## 樣本 1: `184921 / 27 00 / steady`

tail:

- `2321fa94`
- `6f8809ce`
- `a63aa1db`
- `e86fcf4c`

### `n4-n7`

- `n4`: `f -> 0 -> a -> c`
- `n5`: `a -> 9 -> 1 -> f`
- `n6`: `9 -> c -> d -> 4`
- `n7`: `4 -> e -> b -> c`

### 觀察

- 四個 nibble 全部每筆都不同
- 沒有哪一個位置像 `n0` 那樣有清楚骨架
- `n6` 的局部 delta 最平一些，但仍不足以稱為主規律

## 樣本 2: `185451 / 09 0a / steady`

tail:

- `a1669e30`
- `e27e3102`
- `2e097cf8`
- `6c3a68ef`
- `abdf5f55`

### `n4-n7`

- `n4`: `9 -> 3 -> 7 -> 6 -> 5`
- `n5`: `e -> 1 -> c -> 8 -> f`
- `n6`: `3 -> 0 -> f -> e -> 5`
- `n7`: `0 -> 2 -> 8 -> f -> 5`

### 觀察

- 一樣沒有乾淨骨架
- `n7` 出現 `+2 +6 +7 +6`
- `n6` 出現 `+13 +15 +15 +7`
- 這種模式比 `n4/n5` 稍微像有局部結構，但仍屬弱訊號

## 合併結論

`n4-n7` 整體仍然最像 auth-heavy 區。

如果一定要在裡面再分層，目前比較合理的是：

- `n4/n5`: 更高熵
- `n6/n7`: 稍微比較像有局部結構，但仍不足以當主模型

## 目前最實用的 tail 分層

現在可以把 `0x116 tail` 分成：

1. `n0`: 主要 rolling 骨架
2. `n3`: 次要過渡訊號
3. `n1`: 弱過渡訊號
4. `n6/n7`: auth-heavy 內的弱結構區
5. `n4/n5`: 最高熵區

## 對後續的意義

這讓我們之後如果要繼續拆 `0x116 tail`，就不需要平均用力。

更好的優先順序是：

- 先建模 `n0`
- 再拿 `n3` 做輔助
- 最後才看 `n6/n7`

而不是直接把 `n4-n7` 全都當成可以建模的 counter 區。
