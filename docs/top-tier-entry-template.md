# Top-Tier Entry Template

## 目的

找出顶级候选状态出现前的固定前兆。

样本窗口：

- [toyota_seg_IGN_ON_20260312_190101_000.ndjson](../logs/toyota_seg_IGN_ON_20260312_190101_000.ndjson)
- around `1773342250690 -> 1773342250856`

## 原始时序

### `0x116`

- `20 31`
- `2A 3A`
- `43 48`
- `44 46`
- `42 40`
- `3E 3C`

### `0x131 / 0x260`

- `2A 3A` 前后开始进入 `fff4` family 区域
- `43 48`, `44 46`, `42 40` 都稳定落在：
  - `family131 = fff4`
  - `family260 = fff4`
- 到 `3E 3C` 时才开始退到：
  - `family131 = fff4`
  - `family260 = fff0`

### `0x610 / 0xD5`

- `0x610` 在 `1773342250714` 出现 anchor
- `0xD5` 在整个窗口维持 `08000000000000e5`

## `n0` 行为

对应 tail `n0`：

- `20 31` -> `c`
- `2A 3A` -> `0` (`+4`)
- `43 48` -> `c` (`+12`)
- `44 46` -> `0` (`+4`)
- `42 40` -> `4` (`+4`)
- `3E 3C` -> `5` (`+1`)

### 判读

顶级状态前后，`n0` 不是单纯稳态 `+4`，而是：

- 先有一次明显提升到高 phase
- 在顶级区间内仍能看到 `+4`
- 离开顶级区间时开始出现修正步进 `+1`

## 当前最强的进入模板

目前可以把 top-tier candidate 的进入模板先写成：

1. `0x610` 出现事件点
2. `0x116` 先从中高 phase 推到：
   - `20 31`
   - `2A 3A`
3. 接着进入：
   - `43 48`
   - `44 46`
   - `42 40`
4. 同时 `0x131 / 0x260` 稳定落在：
   - `fff4|fff4`
5. 退出时开始转向：
   - `fff4|fff0`
   - phase 下降到 `3E 3C`

## 这代表什么

当前 top-tier state 不是孤立点，而是一段很短的高位平台：

- 进入前有明显 ramp
- 平台上 family pair 稳定
- 退出时 family pair 和 `n0` 都开始松动

## 目前最实用的被动筛选规则

如果之后看新 log，要标记“最接近 security-like state”的顶级候选，可以先找：

1. `0x116` 先出现中高 phase ramp，例如 `20 31 -> 2A 3A`
2. 紧接着出现 `43+ / 44+` 等级高 phase
3. 期间 `0x131 / 0x260` 稳定落在 `fff4|fff4`

只要三者同时出现，就可列为当前最强 top-tier entry。
