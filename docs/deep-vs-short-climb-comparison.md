# Deep vs Short Climb Comparison

## 目的

比較 `0x116` 從 base phase `00 00` 離開後的兩種模式：

- 深爬升
- 短爬升

看它們對應的 `0x131 / 0x260` family state 差在哪。

檔案：

- [toyota_seg_IGN_ON_20260312_190101_000.ndjson](../logs/toyota_seg_IGN_ON_20260312_190101_000.ndjson)

### 判讀

深爬升的特徵很清楚：

1. `0x116` phase 不是只跳幾步，而是長鏈爬升
2. `0x131` 的後半 state 軸從 `0069` 一路展到 `0093+`
3. `0x260` 的 family state 軸同步從 `0069` 展到 `0098+`

也就是：

- phase 深爬升
- family state 也深展開

## 樣本 B: 短爬升

窗口：

- `1773342075803` 附近開始

`0x116` phase：

- `00 00 -> 00 02 -> 02 05 -> 04 06 -> 06 08 -> 08 09 -> 09 09`

`0x131`：

`0x260`：

- `01ff10 ed18`
- `01ff11 ed18`
- `01ff13 ed17`
- `01ff13 ed17`
- `01ff11 ed17`
- `01ff11 ed17`
- `01ff18 ed17`

### 判讀

短爬升的特徵是：

1. `0x116` phase 有爬，但幅度明顯小很多
2. `0x131` 的 family state 軸幾乎不展開，後半維持在 `f986`
3. `0x260` 的 family state 也只有局部擾動，沒有像深爬升那樣整段推進

也就是：

- phase 有提升
- 但 family state 沒有深展開

## 最重要的差異

### 深爬升

- `0x116` 深爬升
- `0x131 / 0x260` family state 深展開

### 短爬升

- `0x116` 只中低度爬升
- `0x131 / 0x260` family state 幾乎不展開

## 目前最重要的工作結論

不是每次 `0x116` 離開 `00 00` 都一樣重要。

更好的判準是：

- 只有當 `0x116` phase 爬升，且 `0x131 / 0x260` family state 也同步深展開時，才是高價值事件帶

## 對你的目標的意義

如果未來要從被動 log 找「可能接近安全門打開」的事件，最值得優先找的是：

- 深爬升 + family state 深展開

而不是只看 `0x116` phase 自己有沒有往上跳。

## 下一步
