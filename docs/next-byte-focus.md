# Next Byte Focus

## 接下來最值得盯的 bytes

現在不需要再平均看整個 8-byte payload 了，可以縮到這幾個位置：

### `0x116`

- `b0`
- `b1`
- `b4-b7`

原因：

- `b0-b1` 最像 phase 區
- `b4-b7` 最像 auth-like tail

### `0x131`

- `b1`
- `b3`

原因：

- `b1` 最像 reset/boundary 主軸
- `b3` 最像背景狀態軸

### `0x260`

- `b2`
- `b4`

原因：

- `b2` 常像 family 內部階段量
- `b4` 和 `0x131.b3` 的耦合最穩

## 接下來不必優先盯的

- `0x115` 全 payload
- `0x260` 的高熵尾端全部 bytes
- `0x131` 的 tail auth 區

這些不是沒價值，只是現在不該先花主要時間。

## 下一輪主線

如果再往下一步收斂，最值得做的是：

1. 把 `0x131.b3` 和 `0x260.b4` 當同一條狀態軸來看
2. 確認 `0x116.b0-b1` 是否總是在這條狀態軸跨邊界時切 phase
3. 最後才回頭看 tail 區是不是有可重放或可預測結構
