# Coupling Map: `0x131` vs `0x260`

## 目標

### 這段的關鍵耦合

最明顯的是：

- `0x131.b3` 和 `0x260.b4` 都走 `3e -> 3d -> 3e -> 3f -> 40 -> 41`

也就是：

- `0x131` 的第 4 byte
- `0x260` 的第 5 byte

在同一事件帶中幾乎走同一條狀態軸。

另外一個重點是：

- `0x131.b1` 有 reset: `37/38/3b/3d/3e -> 00 -> 03/04/06/...`
- `0x260.b2` 沒有 reset，但穩定上升: `42 -> 44 -> 45 -> 47 -> 49 -> 4c -> 4f -> 53 -> ...`

判讀：

`0x260` 不是 `0x131` 的直接複製，但明顯在表達同一個事件帶的另一條狀態軸。

### `0x260`

前 5 bytes:

- `00 fe f2 f9 c7`
- `00 fe fb f9 bc`
- `00 fe fc f9 b2`
- `00 fe fd f9 9a`
- `00 fe fe f9 95`
- `00 ff 01 f9 91`
- `00 fe ef f9 90`
- `00 ff 04 f9 8d`
- `00 fe f6 f9 8d`
- `00 fe e6 f9 8b`

### 這段的關鍵耦合

這段最清楚的不是單 byte 直拷，而是共享背景軸：

- `0x131.b2` 固定為 `f9`
- `0x260.b3` 固定為 `f9`

而且：

- `0x131.b3` 走 `d3 -> c7 -> bc -> a8 -> 9a -> 95 -> 91 -> 90 -> 8d -> 8b -> 8a`
- `0x260.b4` 走 `c7 -> bc -> b2 -> 9a -> 95 -> 91 -> 90 -> 8d -> 8b`

判讀：

`0x260.b4` 很像在跟隨 `0x131.b3` 的背景狀態，只是取樣點不同，或有半步落差。

## 合併結論

目前最像的關係不是：

- `0x260` 等於 `0x131`

而是：

- `0x131` 和 `0x260` 共享同一個 state family
- `0x131` 比較像 boundary-facing frame
- `0x260` 比較像 family snapshot / sync frame

## 目前最值得記住的 byte 關係

### 在 `184921`

- `0x131.b3` <-> `0x260.b4`

### 在 `185451`

- `0x131.b2` <-> `0x260.b3` 都是 `f9`
- `0x131.b3` <-> `0x260.b4` 共享下降背景軸

## 工作假設更新

現在可以把模型再收斂一步：

- `0x131`：邊界 frame
- `0x116`：phase frame
- `0x260`：同家族同步 frame

其中 `0x260` 最重要的觀察位元組是：

- `b4`
- 在某些窗口連 `b3` 也有家族背景軸意義
