## 先讲结论

可以先看到一组**很有用的 control-side 数值**，
也就是我们当前最稳定会看的：

- `0x260`

但要先分清楚：

**这不是已经被验证的“物理方向盘角度”或“前轮转角值”。**

它更准确的定位是：

- 当前最稳的 `control-side command / setpoint` 候选

所以这次我们能先回答的是：

1. 大转向区里，`0x260` 有没有明显大幅变化  
2. 它有没有落在我们旧的 bounded branch 假设范围内

## 对照的旧边界

如果这里说的“之前确认的左右界限”，是我们旧的 bounded branch：

- `[-760, +7981]`

那这次大转向区的结果很清楚：

- **正向一侧没有碰到 `+7981`**
- **负向一侧明显越过 `-760`**

所以这次结果更支持：

**旧的 `[-760, +7981]` 不能再当成真实转向动作里的硬左右界限。**

它最多只能算：

- 较早期的 bounded mapping 假设

不是：

- 实车中真实可观察到的全域左右边界

## 这说明什么

### 1. 这两次大转向区里，`0x260` 确实有明显大幅动作

而且不是小修正，是：

- 负向大约到 `-1508 ~ -1522`
- 正向大约到 `+1003 ~ +1509`

也就是说，大转向区并不是“没有数值变化”，而是：

- **两边都有明显摆动**
- 负向特别深

### 2. 如果拿旧 bounded 假设来比，负向已经超界

旧假设：

- `min = -760`
- `max = +7981`

这次看到：

- `min ≈ -1522`
- 明显低于 `-760`

所以：

- **负向已经超出旧下界**

而正向虽然没有碰到 `+7981`，但这不代表正向界限就是对的，只能说：

- 这次这两段没有把正向推到那么高

### 3. 这更像在说明 bounded branch 不是“真实左右边界”

从这次结果看，比较稳的解释是：

- `[-760, +7981]`
  不是实车里真实动作的物理硬边界
- 它比较像：
  - 旧 control-side bounded mapping 分支的工作假设

## 最诚实的结论

### 我们现在能说的

- 可以从这两次大转向区看到明显的 `0x260` 数值变化
- 这些变化在负向已经超出旧的 `-760` 下界
- 所以旧 bounded branch 不能再当成“大转向是否越界”的可靠实车标准

### 我们现在还不能说的

- 这就是“方向盘物理角度”
- 这就是“前轮真实转角”
- 负向一定等于左转、正向一定等于右转

这些都还需要另外一条**已验证的 steering angle / steering torque 实体信号**来闭合。

## 一句话总结

**可以先看到大转向区里的 control-side 数值，而且它们明确显示负向已经超过旧的 `-760` 边界；所以如果你拿的是之前那组 `[-760, +7981]` 左右界限，这次结果更像是在告诉我们：那组边界不是实车大转向下的真实硬限制。**
